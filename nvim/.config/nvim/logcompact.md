# /logcompact - ログ圧縮プロンプト

logs.mdが長くなったので、以下の方針で圧縮してください：

## 📋 圧縮方針

### ✅ **保持する情報**
- **背景・問題**: なぜその実装が必要だったのか
- **解決手法**: どう解決したのか（核心部分のみ）
- **キーバインド**: どのキーで使うか
- **使用方法**: 実際にどう使うか
- **技術的レガシー**: 将来に活かすべき知見・パターン

### ❌ **削除する情報**
- メリット・結果・技術的価値（現在の実装から明らか）
- 教訓・学び・重要性の説明（不要）
- 感情的表現・装飾（🎉、「世界一」等）
- 詳細なコード例（実装を見れば分かる）
- 表示例・出力例（動作すれば分かる）

## 📅 時期別処理

### **当月分（現在の月）**: 完全保持
- 実装内容・技術仕様・修正箇所等すべて保持
- 最新の作業として詳細な記録を維持

### **前月以前**: 圧縮対象
- 上記方針に従って圧縮
- 重要な実装は個別セクションで詳細保持
- 軽微な機能追加は実装履歴でまとめ

## 🏗️ 構造化ルール

### **重要実装の記録形式**：
```markdown
#### YYYY-MM-DD - 機能名（分類）
**背景**: 問題・必要性
**解決**: 解決手法
**使用方法**: キーバインド・コマンド
```

### **軽微実装の記録形式**：
```markdown
### 機能名実装（YYYY-MM-DD）
- 実装内容の概要
- **使用方法**: キーバインド・コマンド
```

### **技術的レガシー**：
- 将来の実装で参考になる解決パターン
- 同様の問題を避けるための知見
- 革新的な技術手法

### **主要キーマップ一覧**：
- 全体的なキーバインド参照表
- 機能別にグループ化

## 🎯 実行指示

1. 現在のlogs.mdを読み込み
2. 上記方針に従って圧縮
3. 圧縮後のlogs.mdを生成
4. 圧縮率（削減文字数・割合）を報告

**注意**: 履歴として本当に価値のある情報（背景・解決・使用方法）のみを残し、現在の実装状況から推測できる情報は削除すること。
