# タスク管理テスト

## 3状態チェックボックステスト

- [ ] 未完了タスク
- [-] 実行中タスク  
- [x] 完了タスク

## インデントテスト

  - [ ] ネストした未完了
  - [-] ネストした実行中
  - [x] ネストした完了

## 使い方

1. 任意のチェックボックス行にカーソルを置く
2. **Enterキー**を押して状態を循環させる：
   - `[ ]` → `[-]` → `[x]` → `[ ]` → ...

## その他のキーマップ

- `<leader>c` - チェックボックスの追加/削除
- `<leader>1~6` - ヘッダーレベル設定
- `<leader>0` - ヘッダー削除
- `<leader>*` - 箇条書き（*）
- `<leader>-` - 箇条書き（-）

## >63 tree-sitterテスト 
新しい動作：

1～9個のマッチ: a, s, d, f, g, h, j, k, l
10～26個: ホームポジション + 他の小文字
27～52個: + 大文字
53～62個: + 数字
63個以上: 2文字ラベル (aa, as, ad, af...)

確認方法：

nvimを再起動
テキストファイルでfキーを押してみる
ラベルがa, s, d, f, g, h, j, k, lの順で表示されることを確認

設定変更が反映されるように、nvimを再起動してください！どうでしょうか、期待通りの動作になりましたか？
