# index.md Output Format

This reference defines the structure of the `index.md` file generated by the
`vw:index` SKILL. The format is language-neutral: section headings are English
to enable downstream parsing, body content follows the source material's
language.

## Required sections (in order)

```markdown
# <directory-name>

Generated: YYYY-MM-DD | Files: N | Period: YYYY-MM-DD – YYYY-MM-DD

## Overview
(1-3 line narrative describing what this directory contains)

## Timeline
- **YYYY-MM-DD**: event / topic → `source/path.md`
- ...

## Key Topics
- **Topic name**: description → `source/path.md`
- ...

## Decisions
- Decision content → `source/path.md`
- ...

## Open Questions
- Unresolved issue → `source/path.md`
- ...

## Files
- `filename.md` — one-line description
- ...
```

## Rules

### Grounding links
Every item in **Timeline**, **Key Topics**, **Decisions**, and **Open Questions**
MUST end with a grounding link `→ \`path/to/source.md\`` pointing to the source
file(s). If multiple sources, link the primary one and note "(+N more)".

### Length budget
- Target: **10,000 characters or less**
- Priority: coverage over brevity. If exceeding budget, trim lower-priority
  bullets first (keep top 5-10 decisions and open questions).

### Timeline ordering
Chronological, oldest to newest. Entries from the input context's
`## Timeline (date-ordered)` section are pre-sorted; preserve that order.

### Source language
- Section headings (`## Overview`, `## Timeline`, etc.): **English**
- Body content: **follows source files' dominant language**
- Directory name in title: preserve as-is (may be non-ASCII)

### File list section
- Include ALL files from the input context's per-file summaries
- One line per file, format: `` - `filename.md` — one-line description ``
- Description should be extracted from the summary's Overview section

### Optional sections
- **Decisions** and **Open Questions** may be omitted if no content. Do not
  include empty sections.
- If the directory has <5 files, simplified output is acceptable: Overview +
  Files only.

## Input boundary

The LLM generating `index.md` MUST read ONLY the input context file
(`$TMPDIR/vw-index/index_context.md`). It MUST NOT read files from the target
directory directly. This keeps context usage bounded regardless of directory
size.

## Example (abbreviated)

```markdown
# meetings

Generated: 2026-04-23 | Files: 12 | Period: 2026-03-01 – 2026-04-20

## Overview
Project kickoff and weekly sync logs covering the Q2 feature rollout and
vendor coordination.

## Timeline
- **2026-03-01**: Kickoff with PM team → `meetings/2026-03-01_kickoff.md`
- **2026-03-15**: Vendor contract review → `meetings/2026-03-15_vendor.md`
- ...

## Key Topics
- **Q2 roadmap**: Scope frozen after 2026-03-20 review → `meetings/2026-03-20_review.md`
- **Vendor SLA**: Pending legal sign-off → `meetings/2026-04-10_legal.md`

## Decisions
- Launch date moved to 2026-05-15 → `meetings/2026-04-05_replanning.md`

## Open Questions
- Support staffing for post-launch phase → `meetings/2026-04-20_ops.md`

## Files
- `2026-03-01_kickoff.md` — Q2 project kickoff meeting notes
- `2026-03-15_vendor.md` — Vendor contract review session
- ...
```
