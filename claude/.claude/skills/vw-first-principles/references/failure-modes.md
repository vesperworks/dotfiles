# Failure Modes: 第一原理思考の限界と類推が勝つ場面

## Overview

Phase 0（適用判断）と Phase 4（反証）の判断補助。第一原理思考は万能ではなく、マスク自身も「すべてにこの方法では考えられない」と認めている。

## 4 つの失敗モード

| 失敗モード | 説明 | 本スキルでの防御策 |
|-----------|------|------------------|
| 抽象レベルのミス | 論理的に正しい推論でも、抽象化の水準が実用に合わない結論を出す（理論下限 $80/kWh ≠ 明日の価格） | Phase 4-3 で「実用の抽象度に合っているか」を必ず問う |
| 不完全な原理からの推論 | 全前提が真でも、重要な要素が欠けていれば結論が誤る（例: 補助金プログラムを知らずに市場分析） | Phase 2 で数値が取れない項目は「未取得」と明記。でっち上げ禁止 |
| 未知の未知 | 第一原理思考は既知の情報の範囲でしか機能しない。知らない事実は前提に組み込めない | Phase 4-2 で「リストに載っていないカテゴリ」（規制・特許・地政学・運用コスト）を意図的に疑う |
| 高い認知コスト | ルーティン決定への適用は時間の浪費 | Phase 0 で中断判定。中断も正しい選択として扱う |

出典: Commoncog「How First Principles Thinking Fails」 https://commoncog.com/how-first-principles-thinking-fails/

## 類推（Analogy）が勝つ 4 つの場面

Phase 0 の中断判定の根拠。以下に該当したら類推・ベストプラクティス踏襲を推奨して中断する:

1. **時間制約のある意思決定**: 危機対応・即時判断。第一原理の分解をしている時間がない
2. **十分に探索済みの領域**: 長年の蓄積で既に最適化された分野（Idiot Index が 1〜2 倍の領域）。剥がす慣習がもう残っていない
3. **学習の初期段階**: 基礎がない状態でゼロベース構築すると車輪の再発明＋劣化コピーになる。まず確立手法を模倣する
4. **コミュニケーション**: 他者への説明は類推（メタファー）のほうが伝わる。第一原理は分析の道具であって伝達の道具ではない

## 認識論的な注意

> "All knowledge is knowledge from somewhere." — Donna Haraway

「完全に前提ゼロの客観的視点」は認識論的に不可能、という批判が存在する。本スキルの Phase 2 で残す「事実」も観測者の視点・時点に依存する（素材スポット価格は 2012 年時点の値）。**「事実」には必ず時点・条件を添える**ことでこの批判に部分的に応える。

## 反証ループの上限

Phase 4 → Phase 1 の差し戻しは最大 2 周。3 周目に入るのは「未知の未知が多すぎて第一原理の前提リストが収束しない」シグナルであり、適用限界。ユーザーに報告して以下を提案する:

1. 類推アプローチへの切替（確率 50%）: 先行事例の調査に方針転換
2. スコープ縮小（確率 35%）: 問題の一部分だけに第一原理を適用
3. 情報収集フェーズの挿入（確率 15%）: deep-research 等で未知の未知を潰してから再実行
