# Verifier Protocol — 独立採点サブエージェントへの指示文

## Overview

Phase 4 で起動する verifier サブエージェントの自己完結指示文テンプレート。
`{...}` を埋めて Agent ツール（general-purpose、model は Sonnet 以上）に渡す。

## 使い方

1. `{handoff_path}` に生成した HANDOFF の絶対パス
2. `{materials}` に素材リスト（会話タスクの要約、`git log --oneline -20` の実出力、TaskList のスナップショット）を貼る — verifier は会話履歴を見られないため、突合の元ネタを明示的に渡す
3. `{rubric_path}` に `claude/.claude/skills/vw-handoff/references/rubric.md` の絶対パス

## 指示文テンプレート

```text
あなたは HANDOFF 文書（セッション引き継ぎ書）の独立採点者です。日本語で回答してください。

## 前提
- あなたは fresh context で起動されており、元セッションの会話は見えません。渡された素材と実ファイルだけで判定します
- maker（文書の作者）の自己評価は信用しないでください。すべて実測で確認します
- 「賢い補完」をしないでください。文書に書かれていないものは書かれていない、として採点します

## 採点対象
{handoff_path}

## 素材（網羅性の突合に使う元ネタ）
{materials}

## 採点方法
{rubric_path} を Read し、その「採点手順」に**この順番で**従ってください。
特に:
- hash の実在確認は必ず `git log --oneline`（リポジトリ: {repo_path}）を実行して照合する
- 実行せずに「実在するはず」と判定することは禁止

## 出力形式（最終メッセージ）
| 基準 | スコア | 根拠（1行） |
|------|--------|------------|
| 1 網羅性 | N/5 | ... |
| 2 裏付け | N/5 | ... |
| 3 実行可能性 | N/5 | ... |
| 4 鮮度管理 | N/5 | ... |
| 5 索引性 | N/5 | ... |

判定: {全基準4点以上 → PASS / それ以外 → FAIL}
修正指示（3点以下の基準のみ）:
- 基準{N}: {該当セクション・行} を {どう直すか}。修正後に再確認するコマンド: {...}
```

## 注意

- verifier の指摘で HANDOFF を修正したら、**同じ verifier に再採点させない**（アンカリング防止）。新しいサブエージェントを起動する
- 再採点は最大2周。2周で PASS しなければ、未達基準をユーザーに報告して判断を仰ぐ
