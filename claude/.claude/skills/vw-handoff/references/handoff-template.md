# HANDOFF Template

## Overview

vw-handoff の成果物テンプレート。セクションは**この順番で全部**埋める（先頭ほど重要 — 次セッションが最初に読む情報を先頭に置く）。
保存先: `.brain/{project}/handover/{YYYY-MM-DD}-{slug}-HANDOFF.md`

## テンプレート

```markdown
---
date: {YYYY-MM-DD}
author: {モデル名}（vw-handoff）
topic: {slug}
status: active
expires_when: "{この文書が無効になる条件。例: PRP-025 Phase 3 が完了したら / main が push されたら}"
audience: {次の自分 / 別モデル({名前}) / 別人}
---

| タスク | 要点（1行） | hash | 検証状態 |
|--------|------------|------|---------|
| {...} | {成果の中身。調査なら主要発見、実装なら何ができるようになったか} | {abc1234} | {検証コマンド + 結果1行 / 未検証} |

記入ルール:
- **要点列は必須**。ラベルだけの完了行（「調査完了」等）は禁止 — 次セッションが成果を再利用できない
- hash が特定できないものはこの表に入れず、下の「未コミット完了」か「未確認」へ
- 会話内成果（調査・決定）は hash 列に「対象外（会話内成果）」と書き、検証状態に一次記録（エージェント出力・ユーザー回答）を書く

## 記入ルール

- **不明セルは「不明（{何が分かれば埋まるか}）」形式**。空欄・推測は禁止
- 相対日付（昨日、先週）は使わない。すべて絶対日付
- 「注意する」「気をつける」「適切に」を含む行は完成と認めない（3点形式に変換）
- 本文は軽量索引に徹し、長い詳細（調査結果・ログ全文）は別ファイルに置いてパス参照する
