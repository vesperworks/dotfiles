# Financial Visual Grammar (PL / BS / CF)

「会計の地図」「図解総研」系の書籍・記事で使われる財務諸表の視覚文法を、SVG/CSS で再現するための仕様。参照元: note「会社のお金の流れを図解する #会計の地図」「BS（貸借対照表）の図解 #会計の地図」、eurekapu.com のアニメーションB/S図解。

## Overview

財務3表はそれぞれ異なる視覚メタファーで表現する。数値の大小関係を面積・長さで直感的に伝えることを最優先し、テーブルでの表現は補助情報に留める。

## 1. PL（損益計算書）— ウォーターフォールブロック

売上から純利益までを、積み上げ棒の増減（ウォーターフォール）で表現する。

```
売上高 ─┐
        ├─ 売上原価 (▼)
       粗利 ─┐
             ├─ 販管費 (▼)
            営業利益 ─┐
                      ├─ 営業外損益 (▲/▼)
                     経常利益 ─┐
                               ├─ 特別損益・税金 (▼)
                              当期純利益
```

- 各ブロックの高さ = 金額に比例（同一月内は同一スケール）
- 増加要因（売上、営業外収益等）: `--pl-plus`（緑系）
- 減少要因（原価、販管費、税金等）: `--pl-minus`（赤系）
- 最終利益ブロック: `--accent`（強調色）
- 赤字の場合はブロックがゼロ線の下に描画される（マイナス values 対応必須）
- SVG `<rect>` を y 座標の累積オフセットで積み上げる。D3.js 不要、素の SVG 座標計算で十分

## 2. BS（貸借対照表）— 左右対照ブロック（面積比例）

「左側がお金の使いみち（資産）、右側がお金の出どころ（負債・純資産）」という会計の地図の説明をそのまま図解する。

```
┌─────────────┬─────────────┐
│             │  流動負債    │
│  流動資産    ├─────────────┤
│             │  固定負債    │
├─────────────┼─────────────┤
│             │             │
│  固定資産    │  純資産      │
│             │             │
└─────────────┴─────────────┘
   資産合計   =  負債+純資産合計
```

- 左右の全体高さは必ず一致（資産合計 = 負債・純資産合計の会計恒等式を視覚的に保証）
- 各内訳ブロックの高さ = 金額に比例
- 色分け: 流動資産/流動負債 = `--cyan`系、固定資産/固定負債 = `--text-muted`系、純資産 = `--accent`
- 流動比率（流動資産 ÷ 流動負債）が 100% 未満の場合は警告インジケータ（`--danger`の左端ボーダー）を表示
- ホバー時に金額とラベルをツールチップ表示

## 3. CF（キャッシュフロー計算書）— ブリッジチャート

期首現金残高から期末残高までを、営業・投資・財務の3本の増減バーで橋渡しする。

```
期首残高 ──[営業CF]──→ ──[投資CF]──→ ──[財務CF]──→ 期末残高
  ▓▓▓▓        ▲/▼           ▲/▼           ▲/▼         ▓▓▓▓
```

- ウォーターフォールと同じ積み上げロジック（開始/終了は絶対値バー、中間3本は増減バー）
- 営業CF: プラスが基本（`--pl-plus`）、マイナスなら警告色
- 投資CF: マイナスが基本（設備投資等）でも `--text-muted`（中立、必ずしも悪いシグナルではない）
- 財務CF: プラス/マイナスどちらも中立色、大きな借入増加のみ `--warn`

## 4. カラートークン追加（design-system.md への追加分）

`html` スキルの design-system.md の CSS 変数に以下を追加する:

```css
:root {
  --pl-plus: #00ff88;   /* 増加・プラス要因（Hackerプリセット基準、他プリセットは --accent 系で代替可） */
  --pl-minus: #ff5555;  /* 減少・マイナス要因（--danger と共通化可） */
  --bs-current: #00d4ff;   /* 流動資産・流動負債（--cyan と共通化） */
  --bs-fixed: #666666;     /* 固定資産・固定負債（--text-muted と共通化） */
}
```

実装時は `--pl-plus` = `--accent` or 専用緑、`--pl-minus` = `--danger`、`--bs-current` = `--cyan`、`--bs-fixed` = `--text-muted` で流用し、新規変数追加を最小限にする。

## 5. ミニ図解（カレンダーカード内）vs 詳細図解（モーダル）

| 要素 | ミニ図解（カード内、高さ ~120px） | 詳細図解（モーダル、高さ ~400px） |
|------|-----------------------------------|-----------------------------------|
| PL | 5段階に簡略化（売上/原価/粗利/販管費/純利益のみ） | 全項目（営業外・特別損益含む）+ 数値ラベル |
| BS | 左右2ブロックのみ（資産合計 / 負債純資産合計） | 流動/固定/純資産の内訳まで展開 |
| CF | 3本バーの高さのみ（数値なし） | 数値ラベル + 期首・期末残高表示 |

ミニ図解はスキャン性重視（数値を読まなくても増減・健全性が色と形でわかる）、詳細図解は分析性重視（正確な数値を追える）。

## Common Pitfalls

- 赤字/マイナス値のブロックを描画し忘れる（正の値のみを前提にしたスケール計算にしない）
- BS の左右高さがずれる（丸め誤差で資産合計 ≠ 負債+純資産合計になった場合は差額を「調整」ブロックとして明示、無視しない）
- 12ヶ月分のスケールを月ごとに変えてしまう（年間を通じて同一スケールでないと月比較ができない。ただし赤字月と黒字月が混在する場合はゼロ線基準で正負両方向に伸ばす）
