# 採点式・severity 定義

採点は判定役の裁量ではなく、この決定的な式で行う（手計算で検証可能にするため）。

## カテゴリ配点

| カテゴリ | 配点 | 理由 |
|----------|------|------|
| §2 Hygiene | 30 | 機密・個人情報は公開の絶対ブロッカー |
| §1 First Impression | 20 | 初見 30 秒の判断を左右 |
| §3 History | 15 | 公開される履歴の顔 |
| §4 Structure | 15 | 自己説明性 |
| §5 Docs Freshness | 15 | 信頼性（書いてあることが嘘でないか） |
| §6 Taste | 5 | 主観性が高いため最小配点 |
| **合計** | **100** | |

## severity 定義（判定役が最終確定する）

| severity | 定義 | 減点 |
|----------|------|------|
| **Critical** | 公開すると実害がある（機密漏洩・法的不備・動かない手順の公開） | **-10** |
| **Serious** | 実害はないが、初見の信頼を明確に損なう | **-4** |
| **Nitpick** | 直すとより良い。信頼は損なわない | **-1** |

## 採点式

```
カテゴリ得点 = max(0, 配点 - Σ(そのカテゴリの指摘の減点))
総合スコア   = Σ(全カテゴリ得点)
```

- 減点はカテゴリ配点で下限クリップ（Taste で Critical が出ても -5 まで）
- **未検証の仮説は減点しない**（レポートには「未検証」欄で列挙し、検証コマンドを添える）
- 重複統合後の件数で計算する（統合前の Sonnet 申告数ではない）

## push 可否の目安

| スコア | 判定 |
|--------|------|
| 90-100 | そのまま push 可 |
| 70-89 | Serious 対処を推奨（push 自体は可） |
| < 70 または Critical ≥ 1 | **Critical 対処必須**（Critical が 1 件でもあればスコアに関わらずブロック） |

## 判定役の severity 再判定基準

Sonnet の申告 severity は参考値。以下で再判定する:

1. 機密・個人情報の実在が確認できた → Critical に昇格
2. 「動かない」系は検証済みのみ Critical。未検証なら Serious + 未検証マーク
3. Taste カテゴリの申告は原則 Nitpick に降格（§6 の例外条件を満たす場合のみ Serious）
4. 同一根本原因の重複は統合し、減点は 1 回だけ
