# 制約条件
## 言語・思考プロセス（最重要）
- **思考プロセス（Thinking/Chain of Thought）は、必ず日本語で行ってください。英語での思考・独白は厳禁です。**
- 最終的な回答出力もすべて日本語で行ってください。
- 計画（Planning）やウォークスルー（Walkthrough）やタスク（Task）の生成もすべて日本語で行ってください。
- ユーザーが他言語で入力した場合のみ、その言語に合わせて回答してください。

# グローバルルール

## 最重要ルール - 新しいルールの追加プロセス

ユーザーから今回限りではなく常に対応が必要だと思われる指示を受けた場合：

1. 「これを標準のルールにしますか？」と質問する
2. YESの回答を得た場合、CLAUDE.mdに追加ルールとして記載する
3. 以降は標準ルールとして常に適用する

## 提案時のルール

提案を求められた時は、推奨確率を添えて最低3パターン提案してください。

### 選択肢の識別子

- 色の絵文字（🟢🟡🔴🔵 等）は**使わない**。指示や会話で参照しにくいため。
- 代わりに **数字（1, 2, 3…）** を使う。推奨案は先頭に置き「(推奨)」や推奨確率を併記する。
- 例: `1. 推奨 70%: …` / `2. 25%: …` / `3. 5%: …`
- AskUserQuestion でも同様。header や label の先頭を色絵文字にしない。

## /html SKILL 利用ルール

以下の状況では `/html` SKILL を自発的に使用してください（モードは会話文脈から自動判定）：

- 「HTMLで」「ブラウザで」「グラフィカルに」状況・確認事項を整理してほしいと言われた → **status モード**
- 「フロー図」「シーケンス図」「関係図」「Sankey」を作ってほしいと言われた → **diagram モード**
- 画像に「丸つけて」「矢印で示して」「注釈して」と言われた → **annotate モード**
- 画像を「評価して」「採点して」「レビューして」と言われた → **image-review モード**

生成後は必ず `open` でブラウザを自動起動すること（`Bash(open *.html)` 許可済み）。
モード判定が曖昧な場合のみ AskUserQuestion で 1 回確認する。

## コーディング原則

- YAGNI（You Aren't Gonna Need It）：今必要じゃない機能は作らない
- DRY（Don't Repeat Yourself）：同じコードを繰り返さない
- KISS（Keep It Simple Stupid）：シンプルに保つ
- SOLID：オブジェクト指向設計の5原則を守る

## 基本原則

- Explore -> Plan -> Confirm -> Code -> Commit
- **推論の制限**: 記載されていない内容や不明な点について推測せず、必ずユーザーに確認する
- **エスカレーション**: 3回連続で問題に行き詰まった場合は、ユーザーに指示を仰ぐ
- **対話言語**: 日本語でコミュニケーションを行う
- 現在の作業が明確でない場合は、要件定義をヒアリングしてください

## 技術実装の方針

- **公式ドキュメント優先**: ライブラリやフレームワークを使用する際は、最新の公式ドキュメントを必ず参照する
- **設計思想の理解**: 使用する技術の設計思想を理解し、その作法に従って実装を提案する
- **既存コードとの整合性**: プロジェクト内の既存のコーディング規約やパターンに従う
- **ブラウザ操作**: Chrome DevTools MCP を使用する（Playwright は使わない）

## commit規則

- authorを書かない（settings.jsonで設定済み）
- **git push、PRマージなどリモートに影響する不可逆な操作は、実行前に必ずユーザーに確認する**
- **PRマージ時はマージ方法（squash / merge / rebase）を必ずユーザーに確認する**

## 破壊的操作のルール

- **削除は `rm` ではなく `trash` を使う**: macOS のゴミ箱に送られるため復元可能。`rm` は即破棄で取り返しがつかない
- 削除コマンドを pbcopy で渡す際、または提案する際は `trash <path>` 形式に統一する
- 短命な副産物（log ファイル等）でも一律 `trash` を使う（一貫性を保つため）
- 既存の vw-clean SKILL（trash + pbcopy パターン）と整合
