# Plan Mode - リサーチ・計画・管理

計画モードを開始し、リサーチ結果を基に次のアクションを選択できます。

## 使用方法
`/p [調査したい内容や計画したいタスク]`

## 実行フロー

### Step 1: Plan Modeで調査・分析
あなたの要求に基づいて以下を実行します：

1. **徹底的なリサーチ**
   - コードベースの調査
   - 現在の実装状況の確認
   - 問題点や改善点の特定
   - 技術的な実現可能性の検証

2. **詳細な分析レポート**
   - 現状の把握
   - 発見した問題点
   - 解決案の提示
   - 実装に必要な手順

3. **影響範囲の評価**
   - 既存機能への影響
   - リスクの評価
   - 必要な作業量の見積もり

### Step 2: アクション選択

リサーチ完了後、以下の2つから選択してください：

#### 1️⃣ **todo.mdに追加** (`/p add-todo`)
- リサーチ結果を整理してtodo.mdに新しいタスクとして追加
- 優先度の設定
- 具体的なアクションアイテムの作成

#### 2️⃣ **すぐに実行** (`/p execute`)
- Taskに追加
- 必要なファイルの編集
- コードの実装
- テストの作成


## 使用例

### 例1: 新機能の調査
```
/p ユーザー認証機能の改善について調査して
```
→ 現在の認証実装を分析し、改善案を提示

### 例2: バグの原因調査
```
/p worktreeでコミットがmainブランチに行われる問題を調査
```
→ コードを追跡し、問題の原因と解決策を提示

### 例3: アーキテクチャの検討
```
/p マイクロサービス化の可能性を検討
```
→ 現在の構造を分析し、移行計画を提案

## Plan Mode中の振る舞い

- **読み取り専用**: ファイルの編集やコミットは行いません
- **包括的な調査**: 関連するすべてのファイルを調査
- **構造化された報告**: 明確で実用的な情報を提供
- **次のステップの明確化**: 具体的なアクションを提案

## 出力形式

```markdown
# 調査結果レポート

## 📋 要約
[調査内容の簡潔な要約]

## 🔍 詳細分析
### 現状
[現在の実装や状況]

### 発見した問題
[特定された課題や改善点]

### 提案する解決策
[具体的な解決方法]

## 📊 影響評価
- 作業量: [小/中/大]
- リスク: [低/中/高]
- 優先度: [低/中/高]

## 🎯 推奨アクション
1. [具体的なステップ1]
2. [具体的なステップ2]
3. [具体的なステップ3]

---
次のアクションを選択してください：
1️⃣ `/p add-todo` - todo.mdに追加
2️⃣ `/p execute` - すぐに実行
```

## 注意事項

- Plan Mode中はファイルの変更は行われません
- 調査は徹底的に行われますが、実装は選択後に行われます
- todo.mdへの追加は適切な優先度と構造で行われます
